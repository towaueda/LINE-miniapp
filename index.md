3. システム・機能要件（自社開発）
Phase 1からLINEミニアプリとして稼働する仕様です。
ユーザーが個人情報を晒さず、LINE上で完結できる仕組みにします。

技術スタック：
フロントエンド： Next.js + Tailwind CSS（LIFF SDKによるLINE内ブラウザ動作）
バックエンド： Next.js API Routes
データベース： Supabase（PostgreSQL）
認証： LINE認証（LIFF経由）
リアルタイム通信： Supabase Realtime（チャット機能）

A. ユーザー機能（実装済み）
アカウント作成： LINE認証でログイン、ニックネーム登録（本名は運営のみ保持）。
マッチング機能： エリア・空き日程を選択 → 自動で3人組を作成。
グループチャット： マッチングした3人だけの期間限定チャットルーム。
重要： LINE交換不要で待ち合わせが可能。ランチ終了後にルームは消滅。
相互レビュー： 解散後、他メンバーを評価（コミュニケーション、時間厳守、また会いたい度）。
招待コード発行： 一定以上の評価を得た「優良ユーザー」にのみ、友人を招待できる権利を付与（拡散のトリガー）。

B. 管理機能（実装済み）
ダッシュボード： ユーザー数・マッチ数などの統計情報。
ユーザー管理： 登録情報、本人確認、評価スコアの管理。
ブラックリスト機能： ドタキャン、勧誘報告があったユーザーのアカウント停止（BAN機能）。
マッチ管理： グループの作成状況・ステータスの確認・操作。
レビュー管理： ユーザー間の評価内容の閲覧。
お知らせ配信： 協賛ランチやイベント情報の通知配信（全体・個別）。
招待コード管理： 招待コードの発行状況・使用状況の把握。

4. グロース戦略：バイラル係数の最大化
広告費をかけず、ユーザー自身のネットワークで広げる仕組み（Product-Led Growth）を採用します。
「選ばれし者」の演出（初期）：
Phase 1では「完全審査制」とし、通過したこと自体がステータスになるようにブランディング。
招待制による拡散（中期）：
「良い人は、良い友達を知っている」という仮説のもと、高評価ユーザーにのみ招待枠を付与。
「〇〇さんの招待なら安心」という連鎖で、質の高いユーザー層を大阪の主要オフィス街へ広げる。
LINE・SNS活用：
「今日のランチメンバーが最高だった」「会社は違うけど意気投合した」といったUGC（ユーザー投稿）を促進する仕掛けを作る。
LINEのシェア機能を活用した招待導線を整備。
